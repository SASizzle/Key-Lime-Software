# Use Cases

## Purpose

These use cases show how KLS Contact Finder is meant to be used in realistic day-to-day outreach work.

## Use Case 1: First Launch and School List Review

### Goal

Start the app, confirm the environment is working, and begin from the school list.

### Steps

1. Launch KLS Contact Finder.
2. Wait for the local app to open in the browser.
3. Review the dashboard summary and recent scrape area.
4. Open **School List**.
5. Search for a district, city, or school name.
6. Review website status and scrape readiness before taking action.

### Outcome

The user confirms the app is running and has a workable starting list of schools.

## Use Case 2: Find a Website for a School Missing One

### Goal

Give a school record a likely website before scraping.

### Steps

1. Open **School List**.
2. Search for a school with a missing or questionable website.
3. Click **Find Website**.
4. Wait for the discovery result.
5. Review the returned site, status, and notes.
6. Keep the result if it looks correct or edit the school manually if needed.

### Outcome

The school record moves closer to scrape-ready status.

## Use Case 3: Run a Single-School Scrape for an Immediate Need

### Goal

Find likely contacts for one school and review them right away.

### Steps

1. Open **School List**.
2. Confirm the school has a usable website.
3. Click **Run** for that school.
4. Wait for the scrape summary.
5. Open **Quick Review** or **Full Review**.
6. Compare candidate contacts and save the best official contact.

### Outcome

The user gets a reviewable contact set for the school without waiting for a wider batch process.

## Use Case 4: Run a Batch Scrape Across Active Schools

### Goal

Refresh coverage across many schools while continuing other work.

### Steps

1. Open **School List**.
2. Choose a batch scope such as **All Active Schools**.
3. Start the batch scrape.
4. Watch the batch panel for progress.
5. Move to other pages if needed while the job keeps running.
6. Return to the School List or use the global batch banner to monitor status.
7. Cancel only if the run should stop cleanly.

### Outcome

The user updates many school records in one pass without blocking the rest of the workflow.

## Use Case 5: Use Quick Review to Set an Official Contact

### Goal

Make a fast official-contact decision for a school.

### Steps

1. Open **Quick Review** from the School List.
2. Compare the leading candidate contacts.
3. Pick the strongest likely official contact.
4. Add a short reviewer note if needed.
5. Save the review.

### Outcome

The school now has a current verified contact choice for export and follow-up.

## Use Case 6: Use Full Review for Deeper Cleanup

### Goal

Refine a school's contact data beyond the fast-review flow.

### Steps

1. Open the school's **Full Review** page.
2. Filter candidate contacts by search text, status, or available fields.
3. Review notes, completeness, and outreach history.
4. Add a manual contact if the scraper missed someone useful.
5. Save the best official contact.

### Outcome

The school record becomes more accurate and more useful for future outreach.

## Use Case 7: Log Outreach After a Call or Email

### Goal

Keep the contact history current after a real outreach attempt.

### Steps

1. Open **Full Review** for the relevant school or open the **Outreach** page.
2. Select the contact.
3. Add an outreach entry.
4. Choose the method.
5. Add a note and optional outcome.
6. Save the entry.

### Outcome

The app records what happened, when it happened, and what follow-up may be needed next.

## Use Case 8: Import Existing Contacts from a CSV

### Goal

Bring outside contact data into KLS without guessing what will be inserted.

### Steps

1. Open **Import**.
2. Upload a CSV file.
3. Review the preview counts, validation errors, and duplicate summary.
4. Correct the file if needed.
5. Commit the import once the preview looks right.

### Outcome

The user expands the local database while keeping visibility into duplicates and errors.

## Use Case 9: Export a Working Outreach List

### Goal

Produce a filtered file for follow-up work outside the app.

### Steps

1. Open **Export**.
2. Apply search and contact filters.
3. Review the preview table.
4. Choose CSV or XLSX.
5. Download the file.

### Outcome

The user gets a school-level outreach list built from the current saved best-contact records.
